# 🗺️ แผนการพัฒนา Web Photobooth แบบแบ่งเฟส (Development Phases)

แผนงานนี้จัดทำขึ้นโดยสรุปข้อกำหนดจาก `idea.md` ออกเป็น 5 เฟสหลัก เพื่อให้อ่านเข้าใจง่าย แยกสัดส่วนชัดเจน ติดตามความคืบหน้าได้สะดวก และมีระบบทดสอบประสิทธิภาพในทุกขั้นตอน

---

## 🔹 เฟส 1: โครงสร้างพื้นฐานและการทำงานบน Docker (Foundation & Docker Setup)
*เน้นที่การวางฐานโปรเจกต์ให้สามารถรันด้วย Docker และแชร์ออกสู่ภายนอกผ่านอินเทอร์เน็ตได้อย่างปลอดภัย*

1.  **ตั้งค่าโครงการ (Project Setup):**
    *   หน้าบ้าน: Vite + React (TypeScript) + Tailwind CSS v4 + Framer Motion
    *   หลังบ้าน: Node.js (Express) + CORS
2.  **คอนเทนเนอร์ (Dockerization):**
    *   เขียน `Dockerfile` แบบ Multi-stage บิลด์ React แล้วนำไปเสิร์ฟผ่าน Express บนพอร์ต **`3001`** พอร์ตเดียว
    *   เขียน `docker-compose.yml` สำหรับรันคอนเทนเนอร์ชื่อ `y2k-photobooth` และทำการเมานต์โฟลเดอร์ภายนอก (Volumes) ได้แก่:
        *   `server/gallery/` (โฟลเดอร์เก็บภาพจริง)
        *   `server/access.log` (ไฟล์บันทึกกิจกรรมความปลอดภัย)
3.  **การเชื่อมต่อระยะไกล (Remote Access):**
    *   แนะนำและติดตั้งคำสั่งสำหรับรัน **LocalTunnel** (`npx localtunnel --port 3001`) เพื่อให้บุคคลภายนอกเข้าใช้งานจากระยะไกลผ่าน HTTPS ได้อย่างปลอดภัย และอนุญาตให้กล้องมือถือทำงานได้
4.  **การทดสอบประสิทธิภาพ (Performance Testing):**
    *   ทดสอบเวลาในการโหลดหน้าเว็บครั้งแรก (LCP/FCP) ของ Vite Build บน Container
    *   ทดสอบปริมาณการใช้ CPU/RAM ของ Container พอร์ต `3001` และความเร็วในการตอบสนองเมื่อเข้าผ่านเครือข่าย

## 🔹 เฟส 2: ระบบความปลอดภัยขั้นสูงและเหยื่อล่อแฮกเกอร์ (Security & Decoy Honeypot)
*เน้นการป้องกันแกลเลอรีรูปภาพและการจำกัดสิทธิ์ความปลอดภัยในขณะที่เปิดเซิร์ฟเวอร์สู่สาธารณะ*

1.  **การควบคุมสิทธิ์ (Access Control):**
    *   เขียนระบบตรวจสอบสิทธิ์ Admin Token โดยอุปกรณ์แอดมินจะต้องมีรหัสลับที่ถูกต้องใน `localStorage` จึงจะดึงภาพต้นฉบับหรือกดยอมรับเครื่องอื่นได้
    *   บล็อกคำขอทั่วไปไม่ให้เข้าถึงไฟล์ในแกลเลอรีตรง ๆ
2.  **ระบบสร้างโฟลเดอร์ไดนามิก (Dynamic Folder Storage):**
    *   ให้ผู้ใช้กรอกชื่อกลุ่ม/ชื่อเล่นก่อนถ่ายรูป แล้วเซิร์ฟเวอร์จะสร้างโฟลเดอร์เฉพาะในรูปแบบ `server/gallery/[ชื่อกลุ่ม]_[Timestamp]/` เพื่อแยกไฟล์ชัดเจน
3.  **ระบบเหยื่อล่อแฮกเกอร์ (Decoy Honeypot):**
    *   หากผู้ใช้อื่นที่ไม่มีสิทธิ์พยายามแอบสแกนหรือเข้าลิงก์ตรง ๆ เช่น `/gallery` ระบบจะเปลี่ยนเส้นทาง (Redirect) ไปที่ **`server/fake_gallery/`** ซึ่งเก็บภาพมีมล้อเลียน (เช่น Rickroll)
    *   ระบบหลังบ้านจะจับ IP และเขียนระดับภัยเตือนสีแดง `[⚠️ SECURITY WARN]` ลงใน `server/access.log`
4.  **ระบบป้องกันการแกะโค้ด (DevTools Guard):**
    *   หน้าบ้านจะรันสคริปต์บล็อกการคลิกขวา บล็อก F12, Ctrl+Shift+I และย่อขนาดโค้ด (Minify) เมื่อสั่งบิลด์ใช้งานจริง
5.  **การทดสอบประสิทธิภาพ (Performance Testing):**
    *   ทดสอบการหน่วงเวลา (Latency) ในการตอบสนองของ Express Auth Middleware และการกรอง IP สำหรับ Honeypot
    *   ทดสอบการใช้ทรัพยากรเมื่อมีการสแปมยิงขอดูไฟล์รูปภาพใน `/gallery` เพื่อตรวจสอบว่า Honeypot ทำงานได้โดยไม่ทำให้เซิร์ฟเวอร์ล่มหรือค้าง

## 🔹 เฟส 3: ห้องถ่ายรูปและการปรับแต่งทีละสเต็ป (Webcam & Customization Wizard)
*เน้นระบบการถ่ายรูปเว็บแคม และ UI การแต่งรูปสไตล์มินิมอลที่ใช้งานง่าย ไม่รกสายตา*

1.  **กล้องและเว็บแคม (Live Webcam Room):**
    *   ใช้ Web Audio API สังเคราะห์เสียงนับถอยหลังและชัตเตอร์ในตัวเบราว์เซอร์
    *   ทำระบบแสงไฟแฟลชกะพริบสีขาว และกระจกเงา (Mirror)
2.  **ตัวจำลองสีเรียลไทม์ (3D LUT Simulator):**
    *   ทำระบบฟิลเตอร์สีกล้องยอดฮิต (Canon IXY, Kodak Gold, Fuji Superia, Instax) โดยใช้ 3D LUT บนเบราว์เซอร์เพื่อให้ผู้ใช้เลือกดูพรีวิวได้ทันทีและลื่นไหล
3.  **หน้าต่างจัดการแต่งรูปทีละขั้นตอน (Step-by-Step Customization Wizard):**
    *   **ขั้นตอนที่ 1 (Camera Tone):** เลือกโทนสีฟิล์มพื้นฐาน
    *   **ขั้นตอนที่ 2 (Film Adjustments):** เปิด-ปิด และปรับความหนาเม็ดเกรน (Film Grain) และแสงรั่ว (Light Leaks)
    *   **ขั้นตอนที่ 3 (Frame & Label):** เลือกสีกรอบ (ขาว, ดำ, ชมพู, ครีม) และกรอกลายเซ็น/วันที่เขียนมือ
    *   **ขั้นตอนที่ 4 (Soundtrack QR):** วางลิงก์เพลง Spotify/YouTube เพื่อสร้าง QR Code ดีไซน์น่ารักแปะมุมกรอบรูป
4.  **การทดสอบประสิทธิภาพ (Performance Testing):**
    *   ทดสอบอัตราเฟรมเรต (FPS) ของกล้องเว็บแคมพรีวิวในขณะเรนเดอร์ร่วมกับ 3D LUT (Canvas/CSS) เพื่อควบคุมให้การพรีวิวไหลลื่นไม่สะดุด (เป้าหมาย 60 FPS บนมือถือทั่วไป)

## 🔹 เฟส 4: ตัวย้อมสีภาพฝั่งเซิร์ฟเวอร์และแอนิเมชันปริ้นท์ (AI Processing & Interactive Printing)
*เน้นระบบประมวลผลระดับสูงด้วย Python หลังบ้าน และแอนินิเมชันสร้างความพึงพอใจตอนเสร็จสิ้น*

1.  **สคริปต์ย้อมสีภาพด้วย Python (Neural Color DNA Transfer):**
    *   เขียนโค้ด `color_transfer.py` วิเคราะห์ความสว่างและแสง (Adaptive Color Matcher) และปรับย้อมสีภาพในระบบสี LAB (Reinhard Algorithm) ให้เหมือนภาพตัวอย่างของกล้องนั้น ๆ พิกเซลต่อพิกเซล
    *   ตั้งค่าให้ตัวประมวลผลรองรับทั้งการเร่งผ่านการ์ดจอ GPU (GTX 1070 Ti) และสลับกลับมารันบน CPU (Fallback) อัตโนมัติใน Docker หากไม่มีไดรเวอร์ GPU
2.  **การแปลงภาพเคลื่อนไหว (Animated Photo Strip):**
    *   รองรับการถ่ายภาพแบบเคลื่อนไหวสั้น 3-4 เฟรม และเซฟออกมาเป็นแผ่นรูปภาพขยับได้ (ไฟล์ GIF)
3.  **แอนิเมชันเครื่องพิมพ์ (Printer Room):**
    *   รูปภาพเสร็จสิ้นจะค่อย ๆ เลื่อนขึ้นมาจากเครื่องปริ้นท์จำลอง
    *   เพิ่มลูกเล่นลากถูหน้าจอหรือเขย่า (Shake to Develop) เพื่อเร่งให้สีภาพของโพลารอยด์ที่ตอนแรกจาง ๆ ค่อย ๆ พัฒนาตัวคมชัดขึ้นมา
4.  **การทดสอบประสิทธิภาพ (Performance Testing):**
    *   ทดสอบระยะเวลาที่ Python ใช้ในการย้อมสีรูปภาพ (`color_transfer.py`) ทั้งแบบใช้ GPU และ CPU เพื่อเปรียบเทียบหาจุดเหมาะสม
    *   ทดสอบความหน่วงของการส่ง Base64 รูปภาพจาก API และความเร็วในการแปลงและรวมไฟล์ภาพเป็น GIF

## 🔹 เฟส 5: การทดสอบประสิทธิภาพรวมและการปรับปรุงประสิทธิภาพระบบทั้งหมด (Performance Testing & Optimization)
*เน้นการทดสอบระบบแบบ End-to-End ในสภาพแวดล้อมจำลองการทำงานจริง และการทำ Optimization เพื่อลด Resource Usage ของระบบทั้งหมด*

1.  **การทดสอบระบบแบบครอบคลุม (Full End-to-End Testing):**
    *   ทดสอบความเสถียรเมื่อมีการใช้งานกล้องพร้อมกันหลายอุปกรณ์ผ่านอุโมงค์ LocalTunnel HTTPS
    *   ทดสอบการรับส่งไฟล์ (Payload size) ระหว่างหน้าบ้านและหลังบ้าน เพื่อดูความเร็วในการตอบสนอง
    *   ทดสอบขอบเขตความปลอดภัย (Vulnerability Check) เพื่อดูว่าไม่มีช่องโหว่ในการเจาะเข้าแกลเลอรีจริง
2.  **การปรับปรุงประสิทธิภาพระบบ (Optimization):**
    *   **ลดขนาดรูปภาพชั่วคราว (Image Compression):** ก่อนส่งรูปจากเว็บแคมไปประมวลผลที่หลังบ้าน เพื่อประหยัดแบนด์วิดท์ของ LocalTunnel
    *   **การจัดการหน่วยความจำ (Memory Leak Prevention):** เคลียร์ Object URLs และสลายข้อมูล Canvas เพื่อป้องกันหน่วยความจำรั่วไหลบนเว็บเบราว์เซอร์
    *   **ปรับปรุงความเร็วสคริปต์ Python:** ใช้การเขียนโค้ดสไตล์ Vectorization บน NumPy ในสคริปต์ย้อมสีเพื่อให้ทำงานเสร็จในเศษเสี้ยววินาทีแม้รันบน CPU (Fallback)
    *   **การทำ Caching:** ทำการแคชฟิลเตอร์ LUT สำเร็จรูปไว้ในบราว์เซอร์ของลูกค้าเพื่อลดเวลาในการคำนวณซ้ำ
